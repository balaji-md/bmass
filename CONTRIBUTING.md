# Contributing to BMASS

First of all, thank you for considering contributing to BMASS.

Whether you are a systems programmer, AI researcher, Linux developer, security specialist, embedded engineer, clinician, student, technical writer, or simply someone with an interesting idea, your contribution is welcome.

BMASS is an open research project exploring a different way of thinking about computing. We believe the best ideas emerge through open collaboration, rigorous engineering, and respectful discussion.

---

# Our Philosophy

BMASS explores a simple question:

> *What happens when the AI model becomes the primary operating interface rather than another application?*

The project combines deterministic software with adaptive language models while maintaining human oversight, safety, and system integrity.

Not every idea will succeed.

Good engineering means testing assumptions rather than defending them.

---

# Ways to Contribute

You do not need to write code to contribute.

We welcome:

- Code contributions
- Bug reports
- Security reviews
- Performance optimisation
- Documentation improvements
- Testing on different hardware
- Model evaluation
- Benchmarking
- Architecture discussions
- Research references
- Tutorials
- Example use cases

---

# Before You Start

For significant changes, please start a Discussion or open an Issue first.

Large architectural changes are much easier when everyone understands the motivation before implementation begins.

---

# Development Workflow

1. Fork the repository.
2. Create a feature branch.

```bash
git checkout -b my-feature
```

3. Make your changes.

4. Commit with clear commit messages.

```bash
git commit -m "Add shell executor improvements"
```

5. Push your branch.

```bash
git push origin my-feature
```

6. Open a Pull Request.

---

# Coding Principles

We value:

- Simplicity over complexity
- Readability over cleverness
- Security before convenience
- Small focused commits
- Well documented code
- Cross-platform thinking where practical

If something can be simplified, simplify it.

If something can be removed entirely, even better.

---

# Security

BMASS intentionally explores interaction between AI models and operating systems.

Security is therefore a primary concern.

Please report vulnerabilities privately before publishing them.

We especially welcome reviews involving:

- privilege boundaries
- shell execution
- filesystem isolation
- sandboxing
- prompt injection
- model safety
- deterministic safeguards

---

# Design Principles

Current design goals include:

- Local-first
- Privacy-first
- Human-in-the-loop
- Open source
- Hardware agnostic
- Deterministic safety layers
- Modular architecture
- Replaceable models
- Transparent behaviour

---

# What We Appreciate

We value contributions that:

- solve real problems
- reduce unnecessary complexity
- improve documentation
- improve performance
- improve safety
- improve maintainability

Even small improvements matter.

---

# Code of Conduct

Please be respectful.

Challenge ideas.

Do not attack people.

Technical disagreement is encouraged.

Personal hostility is not.

We want BMASS to be a welcoming place for engineers, researchers, students, clinicians, hobbyists, and anyone interested in exploring new computing architectures.

---

# Long-Term Vision

BMASS is an exploration of Probabilistic–Deterministic Human-in-the-Loop (PDHITL) computing.

The long-term objective is to investigate systems in which adaptive models and deterministic software coexist safely, enabling computers that can learn, assist, and improve while remaining under human oversight.

Whether this vision ultimately succeeds or fails, building and evaluating it openly is itself valuable.
