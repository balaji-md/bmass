# Code of Conduct

## Our Commitment

BMASS aims to be a welcoming, respectful, and intellectually rigorous community.

We welcome contributors from all backgrounds, disciplines, and levels of experience.

Everyone should feel comfortable asking questions, proposing ideas, and challenging assumptions in a constructive manner.

---

## Expected Behaviour

We encourage contributors to:

- Be respectful and professional.
- Assume good intentions.
- Focus discussions on ideas rather than individuals.
- Welcome newcomers.
- Support evidence-based discussion.
- Accept constructive criticism.
- Share knowledge openly.
- Help improve the project.

Healthy disagreement is part of good engineering.

---

## Unacceptable Behaviour

The following will not be tolerated:

- Personal attacks
- Harassment or discrimination
- Trolling or deliberately inflammatory behaviour
- Threats or intimidation
- Publishing private information without permission
- Spam or malicious content
- Knowingly introducing malicious code

---

## Engineering Culture

BMASS is an experimental research project.

Ideas should be tested.

Claims should be supported where possible.

Being wrong is acceptable.

Refusing to learn is not.

---

## Enforcement

Project maintainers may edit, remove, lock, or close discussions, issues, or pull requests that violate this Code of Conduct.

Repeated or serious violations may result in removal from the community.

---

## Scope

This Code of Conduct applies to:

- GitHub Discussions
- Issues
- Pull Requests
- Documentation
- Project communications

---

Thank you for helping make BMASS an open, collaborative, and respectful project.
